Frameworks compatibles con Vercel

Descripción rápida de qué significa que un framework esté soportado en Vercel y recomendaciones para desplegar aplicaciones.

¿Qué significa "supported frameworks"?

Un framework compatible en Vercel es aquel para el que la plataforma ofrece presets, optimizaciones de compilación y una experiencia de despliegue casi sin configuración. Esto incluye transformaciones automáticas de los artefactos de build, integraciones con funciones serverless y plantillas de ejemplo.

Frameworks populares que verás en Vercel

  • Next.js (soporte privilegiado y optimizaciones específicas)
  • React (CRA, Vite)
  • Vue / Nuxt
  • Svelte / SvelteKit
  • Astro
  • Remix
  • Gatsby
  • Hugo, Jekyll, Eleventy (sitios estáticos)
  • Y muchos más: Vercel mantiene una lista amplia de frameworks con presets.

Consejos rápidos de despliegue

  1. Usa un repositorio Git (GitHub/GitLab/Bitbucket) y conecta el proyecto a Vercel para despliegues automáticos.
  2. Comprueba el build command y la output directory en la configuración del proyecto si no se detectan automáticamente.
  3. Si usas funciones, colócalas en la carpeta /api o sigue la convención del framework (por ejemplo, rutas en Next.js o Remix).
  4. Activa Previews y Variables de entorno desde el dashboard para pruebas en ramas.

Ejemplo: desplegar con la CLI

npm i -g vercel
cd mi-proyecto
vercel login
vercel --prod

Este flujo es suficiente para la mayoría de proyectos sencillos; para casos avanzados revisa la documentación del preset del framework.

Ventajas de usar Vercel

  • Despliegue rápido desde Git con previews automáticos.
  • Red global CDN + optimizaciones de rendimiento.
  • Funciones serverless y opciones de cómputo (p. ej. Fluid Compute).

¿Qué revisar en la documentación?

Busca el preset del framework, la matriz de compatibilidad de features (ISR, funciones, middleware) y ejemplos oficiales para evitar sorpresas.
